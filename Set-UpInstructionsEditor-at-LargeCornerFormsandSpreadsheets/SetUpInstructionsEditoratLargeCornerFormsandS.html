➤ Division of labor

Editors-in-Chief are responsible for weekly management of the Editors-at-Large. Those in the Admin role are responsible for setting up and maintaining user accounts, spreadsheets, and forms.

➤Weekly Editor in Chief

The weekly EC is responsible for three tasks:

  1. Sending the follow-up email to the previous week’s Eds@L
  2. Tweeting to the current week’s Eds@L
  3. Sending the instructional email to the upcoming Eds@L

Detailed instructions for the weekly EC for both publishing content and for managing the Eds@L are available at https://docs.google.com/a/pressforward.org/file/d/0B9QreyVkdrH1cTE0UUZaVVk4VDQ/edit?usp=sharing

➤ Admin: User Account Creation and Notifications

The Admin is responsible for setting up the form and spreadsheet for each quarter. The process for the transition between forms and spreadsheet pages is still being developed.

The Admin is also responsible for creating user accounts for the incoming Eds@L. This is handled through the “Add Multiple Users” plugin. The plugin will make sure there are no duplicate entries, so there is no need to check ahead of time. Currently we use the plugin by creating a CSV file of the names and email addresses and using the CSV upload function. Check the Add Multiple Users documentation for questions about using the plugin.

We currently use alerts to keep track of new signups. This sends an email to the primary email account whenever the form is filled out. You can find information about setting up notification emails below.

➤ Editing and Embedding Spreadsheets

Once they sign up to be an Ed@L, volunteers are directed to the “Schedule” page of the Editors at Large Corner to confirm their weeks. If dates appear in a column, they signed up for that week.

The schedule sheet is driven by the “[Date] Editors at Large” tab. Because we did not want to display all of the data on the website, we created a “mirror” of the “Responses” sheet that enables us to both exclude and hide columns.

Information is copied over from the “Responses” sheet with the function in A1.

Do be aware that if you sort the items on the “Responses” sheet, they will also sort on the “Editors at Large” sheet. However, hiding columns is isolated to each sheet.

To prepare the sheet for publication, hide the email addresses and any administrative columns. Hide additional columns as desired. Any columns hidden will not appear when the sheet is embedded on a webpage.

➤ Using the spreadsheets and scripts

Once you have a space for Eds@L on your webpage and the appropriate sheets and forms embedded therein, you can use the spreadsheets and the scripts we provide to communicate with your Editors@Large.

➤ Set Up Autoreply

The spreadsheet and form comes with three scripts. The first of these is an auto reply email which is sent to new Eds@L and directs them to the Editors-at-Large Corner of the site.

You can access the script by going to “Tools: Script editor...” and choosing Confirmation AutoReply from the list. (This may be named “Copy of Confirmation Auto Reply” if you have duplicated the sheets.)

This script uses MailApp to send an email as it receives new form submissions.

There are three modifications that you may need to make:

  1.  First, confirm that the column number for the email address correctly matches your spreadsheet. If you have copied ours, ‘3’ should be correct.

  1. Second, craft your auto reply message in the space indicated. You can write the whole message on one line or, for easier readings, you can write across multiple lines using +. Be sure to enclose each line within quotations and end your email with a semicolon after the last quotation mark. “\n” designates a line break and “\n\n” results in space between the lines or paragraphs.

3.   Third, set up the trigger so that the email is sent every time someone submits the form.  

      Go to “Resources: Current Project’s Triggers.” It should inform you that you have no      

      triggers set up. Click “No triggers set up. Click here to add one now” to create a new trigger.

Choose “On form submit” from the drop down and save. Now the confirmation email will be sent with each new submission. You can confirm by volunteering a “test” user.

➤ Customizing the Instructional Email

The second script houses the Instructional Email, which is sent to the Eds@L for the upcoming week and includes links to the instructions on the digitalhumanitiesnow.org site.

To edit the script and set the email text, go again to “Tools: Script Editor” and select “Instructional Email” ( or “Copy of Instructional Email”)

Just as in the confirmation email, you can edit the email text and the subject line as indicated below.

➤ Filtering the spreadsheet by week

When working with the list of Eds@L, the EC is responsible for isolating the editors for the appropriate weeks, sending follow-up emails to the previous week’s editors, tweeting a thank you to the current week’s editors, and sending the instructional email to the upcoming week’s editors.

From the “Responses” sheet, use the “filter” to isolate the editors for the desired week. (Do not sort! Sorting will reorder the schedule sheet.)

Once you activate the filter, a blue button will appear at the top of each column. For the week you wish to isolate, click on the blue button for that column and filter by the date.

This will give you a sheet that only shows the information for the editors-at-large for the selected week.

To turn off the filtering, click the filter button again or re-filter the column, selecting both “(Blanks)” and the date.

➤ Using the scripts to send emails

To send emails using the scripts, select the appropriate button from the top of the spreadsheet from the “Responses” tab. This will launch a dialog box for composing and addressing the email. (The first time you run the script, you will need to authorize it. This enables the script to send emails on your behalf.)

This setup enables the EC to adjust the email text if necessary. However, in most cases the editor should only need to enter the email addresses and select “submit.”

  • Note: When copying and pasting from the column of emails, it isn’t necessary to add commas to separate the addresses. Google handles this on its own.)
