Using the DHNow Theme:

This guide assumes a familiarity with the Wordpress Dashboard. Some sections do refer to making alterations to the theme’s and plugin’s files. PHP knowledge is not necessary as this guide provides instructions on what code to look for and what to replace it with. Any further alterations you wish to make in both style and formatting not covered in this guide do require knowledge of PHP and CSS and this guide points users to which files they can locate the relevant code. If you wish to learn how to make any alterations and changes to the formatting of this theme the Wordpress Codex is an excellent resource.

If you do not know how to access your server to alter Wordpress theme and plugin files, this can be done through the Wordpress Dashboard.

  • To change your Wordpress theme files, select the Editor option under Appearance in the Dashboard menu.
  • To change plugin files, select the Editor option under Plugins in the Dashboard menu.

This guide will also use terminology and taxonomy in reference to DHNow’s set up and operation. Please visit the site for reference to organization and structure. You can change terms as applies to your needs. It is also only directing you to change any Wordpress settings as they pertain to the specific display of this theme. All other settings are your personal preference.

Required Plugins: All of the below can be found in the Wordpress Plugin Directory.

  • Column Posts
  • Smooth Slider
  • Google Fonts

Optional Plugins:

  • Bulk Convert Post Format
  • If you already have an existing Wordpress site this plugin allows you to quickly convert post formats in bulk. Necessary to properly display an Editors’ Choice category.
  • Categories to Tags Converter Importer
  • If you already have an existing Wordpress site that used tags as its main organizing structure this plugin assists with converting the tags to categories to map existing content to the theme’s structure.
  • Clean Archives Reloaded
  • This plugin is used to create an archives page that lists all Editors’ Choice posts for authors to have a stable url to use to cite their publication on your site.

DHNow’s theme is based on the Wordpress TwentyTwelve theme which must be installed in order for this theme to function.

Preliminary Steps

  1. Download and activate TwentyTwelve for DHNow theme as well as all required plugins.
  2. Create your categories. This theme works best with a minimum of 4-5 categories:
  • Editors’ Choice (or whatever category you wish to be displayed in the slider on the main page). It is recommended that whatever you name your category you should make the slug of this category ‘featured.’
  • News (or whatever general name you wish to use). This facilitates users being able to subscribe to all the News Sub-Categories en masse via RSS if you wish and is necessary for some formatting to display publically.
  • Create this category before any sub-categories as it is imperative that this category have a lower category ID than your sub-categories.
  • 2-3 News Sub-Categories, ie Resources, Job Opportunities, CFPs (or whatever categories you wish to have display in the columns on the main page)
  • This theme displays best with either an even number or multiple of three News sub-categories (2, 3, 4, 6, 8, 9, etc)
  1. Note somewhere the category ID number for all your created categories.
  • This can be found by hovering over the Category name and locating the ID number in the URL that displays at the bottom of the screen.

 

Creating & Formatting the Front Page

  1. Create a page titled “Home,” or whatever you choose. The page title will not actually display on the front end.
  2. In the content box type: [columnpost]
  3. Set the page template to: Front Page Template
  4. In the Reading option under Settings in the Dashboard menu:
  1. Select your Front Page to display a static page
  2. Select your “Home” page from the pull down menu
  3. You do not need to select anything for your posts page.

Header:

  • This theme is designed to display a header image in place of the Site title.
  • Install your image in the Header option under Appearance in the Dashboard menu.
  • If you select to “Display header text with your image” the theme will display your site’s tagline (determined in the General option under Settings in the Dashboard menu) underneath the image.
  • If you wish to change this configuration it is generated by the header.php file in the theme’s folder.

Navigation Bar:

It is advised to create your Navigation bar by creating a custom Main Menu in the Menus option under Appearance in the Dashboard menu.

Having Only One Category Display in the Smooth Slider:

If you did not create your Editors’ Choice category with the slug ‘featured’ you can either go back and change it or if you wish to keep the Name and the Slug of the category the same, follow these steps.

  1. Open the theme’s front-page.php file.
  2. On line 27 (or there abouts) find this line of code:

<?php if ( function_exists( 'get_smooth_slider_category' ) ) { get_smooth_slider_category('featured'); } ?>

  • Note: if you are editing this file via the Wordpress Dashboard Editor lines are not numbered
  1. Change ‘featured’ to the slug of your Editors’ Choice category.

Creating a Citeable Archive Page:

Please note that this is an alteration of the plugin that you download from the Wordpress Plugin Directory and will have to be repeated (and/or adjusted) when you update the plugin. This hack is only if you want just one category to appear on the archives page.

  1. Install and Activate the Clean Archives Reloaded Plugin
  2. Adjust the settings under the Dashboard Appearances menu to your preference.
  3. In the clean-archives-reloaded.php file:
  • Find the GetPosts() function.
  • Replace

$rawposts = $wpdb->get_results( "SELECT ID, post_date, post_date_gmt, comment_status, comment_count FROM $wpdb->posts WHERE post_status = 'publish' AND post_type = 'post' AND post_password = ''" );

  • With

$rawposts = get_posts( array( ‘numberposts’ => -1, ‘category’ => ‘#’ ) );

Replace the # with the ID of the category you wish to display

  1. On a new Wordpress page in addition to whatever else you wish to write insert the shortcode: [cleanarchivesreloaded]

Publishing an Editors’ Choice Post

This theme is designed to include a featured image for each Editors’ Choice post. While it is not necessary it will affect the display and formatting for the Smooth Slider, the Archive page for this category, and the individual post’s page. You can fix any formatting issues that arise from not publishing with a Featured Image in the theme’s style.css file.

Required Elements:

  • Categories: Editors’ Choice
  • Featured Image: Assign a featured image
  • Format: Editors’ Choice
  • Smooth Slider: Check only the box to ‘Add this post/page to Smooth Slider’

Optional Functionality:

Custom Fields: (this box does not automatically display when publishing a new post on a fresh Wordpress install. To display it, select the Custom Fields box under the dropdown Screen Options tab on the top right of the screen)

  1. Changing the author name:
  • For the first post click ‘Enter new’ in the Custom Fields box and exactly type in the Name field: guest-author
  • This only has to be done once and for all subsequent posts the ‘guest-author’ name can be selected from the dropdown menu
  • Type author’s name in the Value field
  • Click ‘Add Custom Field’
  1. Adding Editor-in-Chief and Editors-at-Large: 
  • Follow the instructions above except for the first post (and to select for subsequent posts from the dropdown) exactly type in the Name field (separately): editor-in-chief and editors-at-large
  • In order for this custom field to display, there must be a value for the editor-in-chief custom field. If there is a value for the editor-in-chief custom field, there must also be a value for the editors-at-large custom field.

Note: If you wish to change the language or create additional custom field displays, that will be explained in greater detail below.

Changing the Post Type Format display for ‘Editors’ Choice’:

If you wish to use a different term than ‘Editors’ Choice’ but wish to continue to have the post type renamed it can be changed in the theme’s functions.php file.

  • In the rename_post_formats and live_rename_formats functions, just replace all instances of ‘Editors’ Choice’ with the term you prefer.

Using the Bulk Convert Post Format Plugin:

This plugin is fairly straightforward to use but the theme has superficially renamed the post format type for Editors’ Choice pieces. When using this plugin and you wish to convert existing posts to Editors’ Choice post types, select Image post types from the dropdown menu.

Displaying/Altering Custom Field Content

The DHNow theme has two Custom Fields already built in: The Editor-in-Chief and the Editors-at-Large. You will need to alter the language to reflect your site and your needs. How to do this will be explained in a bit.

To have the content displayed on an Editors’ Choice post:

  • In a new post select to display the Custom Fields from the Screen Options tab in the top right hand corner.
  • In the Custom Fields box
  • Select Enter New
  • In the Name box type: editor-in-chief
  • In the Value box type: your name
  • Repeat with editors-at-large in the Name box and your editors-at-large in the Value box
  • You only have to Enter New fields once. Afterwards, they will be available from a drop-down menu.

The Editor-in-Chief/Large paragraph will not display on any post unless there is content in the Value box of the editor-in-chief custom field. If there is content in the editor-in-chief value box there has to also be content in the editors-at-large value box. You can choose to not feature the editors-at-large by changing the content-image.php file.
